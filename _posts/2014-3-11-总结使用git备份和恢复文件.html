    本人接触git有近4个月时间了，断断续续的用它上传又下载源码。 因为是新手，毫无疑问，遇到过很多困惑。下面就讲述一下在使用git上传和恢复代码的时候遇到的问题。 如果你是新手，不自觉喜欢手动删除本地源码，那么本文或许会对你有较大帮助，如果你是高手，不足指 出多多请教，嘿嘿。 

     如果你使用了git，那么你肯定有了github帐号，也在该网站上建立了代码库（我称之为“远程代码库”）。
然后你会在本地创建一个目录存放代码（这些非隐藏的文件我称它们“本地代码库”）；当你在终端使用了“git init”
命令之后就创建了一系列隐藏的文件，这些文件在使用命令“git commit..”之后就是本地代码的副本（我称它
为“中转代码库”），之所以称它为“中转代码库”是因为，这些隐藏的代码充当“本地代码库”和“远程代码库”之间的
桥梁，“本地代码库”并不能直接与“远程代码库”进行相互复制和替换。好了有了这三个概念之后，我就可以很简单的
告诉大家这三者之间是如何工作的。

注意：第5个情况不建议使用，特别是在协同开发的情况下，免得丢失他人已保存的数据。

    


	文件传递方向
	使用命令




1.“本地代码库”=>“中转代码库”
git add . + git commit




2.“中转代码库”=>“远程代码库”
git push 




3.“远程代码库”=>“中转代码库”
git fetch




4.“中转代码库”=>“本地代码库”
git stash




5.“远程代码库”=>“本地代码库”
git pull 或git fetch + git merge
