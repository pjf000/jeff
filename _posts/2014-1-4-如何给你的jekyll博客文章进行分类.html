    有时候我们可能会遇到一些奇奇怪怪的需求，比如发一个邀请链接（包含你的id）给你的好友，你的好友通过这个链接下载安装包安装完后你的邀请人数就加1，这时候就希望用链接下载的apk可以根据链接的参数给apk一个“标记”，所谓标记就是
     1.要么修改apk名字
     2.要么在apk里写一些东西
    第一种方案太low了，谁都看得出来。第二个方案看似可行，问题是apk是一个编译好的安装包，怎么往里写东西呢？写了东西会不会影响安装呢？

	事实上，每个apk都是一个压缩包，把他后缀改成zip试试。我用的后台脚本语言是php，有现有的方法对一个zip进行一些操作，其他脚本语言也肯定会有，下面我就介绍下我的做法：
	1.生成一个临时zip文件
	2.读取apk安装包，添加到zip
	3.根据get参数制作json文件，添加到zip
	4.输出给客户端
	直接上代码吧：
open($tmpfile); // 添加文件 $zip->addFromString($ef, json_encode($data)); // 关闭zip $zip->close(); // 下载文件 // header("Content-Type: application/apk"); header('Content-Type: application/vnd.android.package-archive'); header("Content-Length: " . filesize($tmpfile)); header("Content-Disposition: attachment; filename=\"{$apkname}\""); // 输出二进制流 readfile($tmpfile); // 删除临时文件 unlink($tmpfile); exit; ?> 值得注意的地方是 //header("Content-Type: application/apk"); header('Content-Type: application/vnd.android.package-archive'); 如果用 header("Content-Type: application/apk"); 来告诉浏览器接收到的文件是可安装apk，很多手机的自带浏览器是不识别的，会出现下载后点击显示“无法解析文件”等错误。 用UC,QQ浏览器测试均没问题，这个问题困扰了我很久，一直以为是重写apk的时候把apk写坏了。 后来才发现用 header('Content-Type: application/vnd.android.package-archive'); 可以让普遍浏览器识别出下载的文件是安装包。 app安装后获取 META-INF/EXTENDS.JSON 文件里的数据传给后台，整个流程就结束了。 ok~！就这么多。
